# ReproChecker 数据库设计

> 版本: v0.1 | 日期: 2026-05-28

## 存储策略

- **主存储**：SQLite（单文件，零配置）
- **数据库文件**：`~/.reprochecker/checks.db`（用户级）
- **报告输出**：HTML/PDF 文件，保存在 `~/.reprochecker/reports/`

---

## Schema

### checks — 检验记录主表

```sql
CREATE TABLE checks (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_url        TEXT NOT NULL,                   -- GitHub 仓库 URL
    pdf_path        TEXT,                            -- 论文 PDF 路径（可选）
    repo_name       TEXT,                            -- 仓库名 e.g. "user/repo"
    commit_hash     TEXT,                            -- clone 时的 commit

    -- 仓库分析结果
    framework       TEXT,                            -- pytorch / tensorflow / jax / unknown
    python_version  TEXT,                            -- 检测到的 Python 版本要求
    has_dockerfile  BOOLEAN,                         -- 是否有 Dockerfile
    has_requirements BOOLEAN,                        -- 是否有 requirements.txt
    entry_script    TEXT,                            -- 识别到的入口脚本 e.g. "train.py"
    has_seed        BOOLEAN,                         -- 是否设置了随机种子

    -- 环境信息
    env_method      TEXT,                            -- docker / conda / venv / manual
    env_setup_log   TEXT,                            -- 环境搭建日志
    installed_packages TEXT,                         -- pip freeze 结果（JSON）

    -- 运行结果
    run_command     TEXT,                            -- 实际运行的命令
    run_status      TEXT NOT NULL DEFAULT 'pending', -- pending / running / success / failed / timeout
    exit_code       INTEGER,
    stdout          TEXT,                            -- 标准输出（截断）
    stderr          TEXT,                            -- 标准错误
    start_time      TIMESTAMP,
    end_time        TIMESTAMP,
    duration_sec    REAL,

    -- 资源使用
    peak_gpu_mem_mb REAL,
    peak_cpu_percent REAL,
    peak_ram_mb     REAL,
    model_params    INTEGER,                         -- 模型参数量
    model_size_mb   REAL,                            -- checkpoint 文件大小
    inference_ms    REAL,                            -- 单张推理耗时 (ms)

    -- 评分
    overall_score   REAL,                            -- 总分 0-100
    grade           TEXT,                            -- A/B/C/D/F
    metric_score    REAL,                            -- 指标复现分
    env_score       REAL,                            -- 环境复现分
    code_score      REAL,                            -- 代码质量分

    -- 元数据
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    notes           TEXT,
    report_path     TEXT                             -- 生成的报告文件路径
);

CREATE INDEX idx_checks_repo ON checks(repo_url);
CREATE INDEX idx_checks_grade ON checks(grade);
CREATE INDEX idx_checks_created ON checks(created_at);
```

### paper_results — 论文报告结果

```sql
CREATE TABLE paper_results (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    check_id        INTEGER NOT NULL REFERENCES checks(id) ON DELETE CASCADE,
    table_caption   TEXT,                            -- 表格标题
    method_name     TEXT,                            -- 方法名 e.g. "Ours"
    metric_name     TEXT NOT NULL,                   -- 指标名 e.g. "accuracy"
    metric_value    REAL NOT NULL,                   -- 数值
    is_best         BOOLEAN DEFAULT FALSE            -- 是否为论文声称的最佳结果
);

CREATE INDEX idx_paper_results_check ON paper_results(check_id);
```

### actual_results — 实际运行结果

```sql
CREATE TABLE actual_results (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    check_id        INTEGER NOT NULL REFERENCES checks(id) ON DELETE CASCADE,
    metric_name     TEXT NOT NULL,                   -- 指标名
    metric_value    REAL NOT NULL,                   -- 实际数值
    step            INTEGER,                         -- 训练步数（可选）
    epoch           INTEGER,                         -- epoch（可选）
    timestamp       TIMESTAMP
);

CREATE INDEX idx_actual_results_check ON actual_results(check_id);
```

### comparisons — 对比结果

```sql
CREATE TABLE comparisons (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    check_id        INTEGER NOT NULL REFERENCES checks(id) ON DELETE CASCADE,
    metric_name     TEXT NOT NULL,
    paper_value     REAL,                            -- 论文报告值
    actual_value    REAL,                            -- 实际运行值
    absolute_error  REAL,                            -- 绝对误差
    relative_error  REAL,                            -- 相对误差 (%)
    within_tolerance BOOLEAN,                        -- 是否在容差范围内
    tolerance_band  TEXT                             -- <1% / 1-5% / 5-10% / 10-20% / >20%
);

CREATE INDEX idx_comparisons_check ON comparisons(check_id);
```

### training_curves — 训练曲线数据

```sql
CREATE TABLE training_curves (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    check_id        INTEGER NOT NULL REFERENCES checks(id) ON DELETE CASCADE,
    metric_name     TEXT NOT NULL,                   -- e.g. "loss", "accuracy"
    step            INTEGER NOT NULL,
    value           REAL NOT NULL,
    epoch           INTEGER,
    timestamp       TIMESTAMP
);

CREATE INDEX idx_curves_check ON training_curves(check_id);
CREATE INDEX idx_curves_metric ON training_curves(check_id, metric_name);
```

## 存量预估

| 表 | 单行大小 | 100 次检验预估 |
|----|----------|---------------|
| checks | ~5 KB | ~0.5 MB |
| paper_results | ~100 B × 20 行 | ~0.2 MB |
| actual_results | ~80 B × 500 行 | ~4 MB |
| comparisons | ~150 B × 20 行 | ~0.3 MB |
| training_curves | ~60 B × 5000 行 | ~30 MB |

**总计**：100 次检验约 35 MB，SQLite 轻松应对。

## 查询场景

### 常用查询

```sql
-- 最近的检验记录
SELECT id, repo_name, grade, overall_score, run_status, created_at
FROM checks ORDER BY created_at DESC LIMIT 20;

-- 某次检验的完整对比
SELECT c.metric_name, c.paper_value, c.actual_value,
       c.relative_error, c.tolerance_band
FROM comparisons c
WHERE c.check_id = 42;

-- 某次检验的训练曲线
SELECT step, value FROM training_curves
WHERE check_id = 42 AND metric_name = 'loss'
ORDER BY step;

-- 可复现性最好的仓库
SELECT repo_name, overall_score, grade
FROM checks WHERE run_status = 'success'
ORDER BY overall_score DESC LIMIT 10;

-- 同一仓库的多次检验对比
SELECT id, commit_hash, grade, overall_score, created_at
FROM checks WHERE repo_url = 'https://github.com/user/repo'
ORDER BY created_at DESC;
```
