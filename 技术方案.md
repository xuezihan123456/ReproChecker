# ReproChecker — 可复现性检验器 技术方案

> 版本: v0.1 | 日期: 2026-05-28

## 1. 项目概述

### 1.1 定位

学术论文可复现性自动检验工具。输入 GitHub 仓库 URL + 论文 PDF，自动完成：clone → 安装依赖 → 运行实验 → 解析论文结果 → 全维度对比 → 输出可复现性报告。

### 1.2 解决的问题

- 论文报了 92% accuracy，clone 下来跑只有 85%，不知道是环境问题还是论文注水
- 复现一篇论文平均耗时数天到数周，大量时间花在环境配置上
- 没有标准化的可复现性评估体系
- 审稿人很难验证论文结果的真实性

### 1.3 核心价值

| 特性 | 说明 |
|------|------|
| 全自动 | 输入 URL + PDF，等待结果即可 |
| 全维度 | 指标、训练曲线、模型大小、推理速度 |
| 标准化评分 | A/B/C/D/F 等级，一目了然 |
| 可重复 | 同样的输入，同样的结果 |

## 2. 技术栈

| 层 | 技术选型 | 理由 |
|----|----------|------|
| 核心逻辑 | Python | ML 生态一致 |
| PDF 解析 | PyMuPDF + LLM API | 文本提取 + 结构化理解 |
| 仓库分析 | AST 解析 + 文件模式匹配 | 检测框架、入口、配置 |
| 环境管理 | Docker / conda / venv | 优先 Docker 保证一致性 |
| 实验运行 | subprocess + 资源监控 | 捕获输出 + GPU/CPU 监控 |
| 报告生成 | Jinja2 + WeasyPrint | HTML/PDF 报告输出 |
| 存储 | SQLite | 检验记录持久化 |
| CLI | Typer | 命令行入口 |

## 5. 详细设计

### 5.1 仓库分析（repo/analyzer.py）

自动检测项目信息：

| 检测项 | 方法 | 输出 |
|--------|------|------|
| 框架 | AST 扫描 import 语句 | PyTorch / TensorFlow / JAX / 其他 |
| Python 版本 | setup.py / pyproject.toml / classifiers | ">=3.8" |
| 依赖文件 | 文件存在性检测 | requirements.txt / setup.py / environment.yml |
| 入口脚本 | 关键字匹配（train/main/run） | train.py / main.py / run.sh |
| 配置文件 | 扫描 *.yaml / *.toml / *.json | config.yaml 等 |
| 数据集 | 扫描数据路径 + 下载脚本 | data/ / dataset/ / download.sh |
| 预训练权重 | 扫描 checkpoint 路径 + .pt/.pth/.ckpt 文件 | checkpoints/ 等 |
| 随机种子 | AST 扫描 seed/random/np.random | 是否设置了种子 |

### 5.2 环境搭建（repo/env_builder.py）

优先级策略：

```
1. Dockerfile 存在？
   ├─ Yes → docker build + docker run（最可靠）
   └─ No ↓
2. environment.yml 存在？
   ├─ Yes → conda env create
   └─ No ↓
3. requirements.txt 存在？
   ├─ Yes → pip install -r requirements.txt
   └─ No ↓
4. setup.py / pyproject.toml 存在？
   ├─ Yes → pip install -e .
   └─ No ↓
5. 提示用户手动配置
```

### 5.3 实验运行（runner/executor.py）

- 自动推断入口命令：`python train.py` / `bash run.sh` / 自定义
- 超时保护：默认 4 小时，可配置
- 输出截断：保留前 200 行 + 后 800 行
- 随机种子注入：如果代码未设置种子，自动注入 `--seed 42`
- 断点续跑：如果支持 `--resume` 参数，可从 checkpoint 继续

### 5.4 PDF 解析（pdf/llm_parser.py）

**Step 1 — 文本提取**（PyMuPDF）：
- 提取全文文本
- 识别表格区域
- 识别图表 caption

**Step 2 — LLM 结构化提取**：

```python
prompt = """
你是一个学术论文结果提取专家。从以下论文文本中提取所有报告的实验结果。

要求：
1. 提取所有表格中的数值结果
2. 提取正文中的关键数值
3. 识别最佳结果（论文声称的 SOTA）
4. 识别使用的数据集和模型

返回 JSON：
{
  "dataset": "CIFAR-10",
  "model": "ResNet-50",
  "tables": [
    {
      "caption": "Table 1: Comparison with state-of-the-art",
      "headers": ["Method", "Accuracy", "F1", "Params"],
      "rows": [
        {"Method": "Baseline", "Accuracy": 89.2, "F1": 87.5, "Params": "25.6M"},
        {"Method": "Ours", "Accuracy": 92.1, "F1": 90.3, "Params": "26.1M"}
      ]
    }
  ],
  "claimed_best": {
    "accuracy": 92.1,
    "f1": 90.3,
    "method": "Ours"
  }
}
"""
```

### 5.5 全维度对比

#### 5.5.1 最终指标对比

| 误差范围 | 评级 | 说明 |
|----------|------|------|
| < 1% | 完全复现 | 可忽略的随机性 |
| 1-5% | 基本复现 | 正常的随机波动 |
| 5-10% | 部分复现 | 可能有环境差异 |
| 10-20% | 难以复现 | 需要检查原因 |
| > 20% | 无法复现 | 计文可能有问题 |
| 无法运行 | 完全失败 | 代码/环境问题 |

#### 5.5.2 训练曲线对比

- 使用 DTW（Dynamic Time Warping）计算曲线相似度
- 检查收敛趋势是否一致
- 检查是否出现过拟合/欠拟合

#### 5.5.3 模型大小对比

- 参数量：`sum(p.numel() for p in model.parameters())`
- 文件大小：checkpoint 文件体积
- FLOPs：如果可检测，用 thop/torchprofile 计算

#### 5.5.4 推理速度对比

- 单张图片推理耗时（batch_size=1）
- 批量推理吞吐量（batch_size=32）
- 注意：硬件差异大，仅作参考

### 5.6 可复现性评分

```
总分 = 指标复现 × 50% + 环境复现 × 20% + 代码质量 × 30%

等级：
  A (90-100): 完全可复现
  B (75-89):  基本可复现
  C (60-74):  部分可复现
  D (40-59):  难以复现
  F (0-39):   无法复现
```

## 6. CLI 命令设计

| 命令 | 说明 |
|------|------|
| `repro check <url> --pdf paper.pdf` | 完整检验流程 |
| `repro check <url>` | 仅运行代码（不解析 PDF） |
| `repro list` | 列出历史检验记录 |
| `repro show <id>` | 查看检验详情 |
| `repro report <id> [--format html/pdf]` | 生成/导出报告 |
| `repro compare <id1> <id2>` | 对比两次检验结果 |

## 9. 风险与对策

| 风险 | 对策 |
|------|------|
| 项目结构千差万别 | 支持手动指定入口命令和配置 |
| 依赖安装失败 | 提供手动干预接口，记录失败原因 |
| 数据集太大/需申请 | 跳过数据集下载，提示用户提供 |
| PDF 表格解析不准 | LLM + 正则双保险，支持手动修正 |
| 论文格式差异大 | 支持多种 LLM prompt 模板 |
| GPU 硬件差异 | 明确标注硬件信息，推理速度仅作参考 |
| 运行时间过长 | 超时保护 + 可配置最大运行时间 |
